# 调试日志指南

已为项目的所有关键操作添加了详细的日志记录，方便快速定位问题。

## 日志级别说明

- **`logger.info()`**: 重要操作开始/完成，如搜索开始、下载完成
- **`logger.debug()`**: 详细步骤，如 URL 构建、配置参数
- **`logger.warning()`**: 需要注意的情况，如降级到备用方案
- **`logger.error()`**: 错误发生，带有异常类型和堆栈信息

## 查看实时日志

在 Ubuntu 上运行 NoneBot 服务时，使用以下命令查看日志：

```bash
# 查看实时日志
journalctl -u nonebot -f

# 查看最近 100 行日志
journalctl -u nonebot -n 100

# 查看特定时间范围的日志
journalctl -u nonebot --since "2026-03-14 10:00:00" --until "2026-03-14 11:00:00"

# 搜索特定关键字
journalctl -u nonebot | grep "\[搜索\]"
journalctl -u nonebot | grep "\[下载\]"
journalctl -u nonebot | grep "\[上传\]"
journalctl -u nonebot | grep "\[存档\]"
```

## 排查常见问题

### 问题 1: /search 命令无响应

查找日志：
```bash
journalctl -u nonebot | grep -A5 "\[搜索处理\]"
```

**如果看到**：
- `[搜索] 直连 IP 连接失败，自动降级` → IP 被封，自动切换到 DNS
- `[搜索] 使用标准 DNS 重试搜索` → 成功切换
- `[搜索] 搜索失败: xxx` → DNS 也被封，需要配置代理

### 问题 2: 下载失败

查找日志：
```bash
journalctl -u nonebot | grep -A20 "\[下载处理\]"
```

**檢查步驟**：
1. 搜索是否成功找到目标 → `[下载处理] 找到目标:`
2. 存档是否成功解析 → `[存档] 成功获取下载链接`
3. 文件是否成功下载 → `[下载] 下载成功，大小:`
4. 群文件是否上传成功 → `[上传] 群文件上传成功`

### 问题 3: 群文件上传超时

查找日志：
```bash
journalctl -u nonebot | grep -B5 -A5 "\[上传\]"
```

**可能原因**：
- `[流上传] 上传数据块失败` → 网络不稳定，尝试流上传失败后会使用本地路径重试
- `[上传] 群文件上传失败` → NapCat WebSocket 响应超时，与机器人连接问题有关

### 问题 4: 自动 DNS 降级不工作

查找日志：
```bash
journalctl -u nonebot | grep "降级"
```

**应該看到**：
```
[搜索] 直连 IP 连接失败，自动降级到标准 DNS
[搜索] 使用标准 DNS 重试搜索
[搜索] 标准 DNS 搜索成功
```

如果没有看到 `DNS 成功`，说明 DNS 层也被封，需要配置代理。

## 配置代理以使用日志导调

如果标准 DNS 也失败，需要配置 `.env`：

```env
# 必须的 Cookie
EHENTAI_IPB_MEMBER_ID=your_id
EHENTAI_IPB_PASS_HASH=your_hash

# 代理配置
EHENTAI_PROXY=http://127.0.0.1:7890

# 关闭直连 IP 模式（强制使用代理）
EHENTAI_ENABLE_DIRECT_IP=false
```

配置后查看日志：
```bash
journalctl -u nonebot | grep "代理"
```

## 提交问题报告时

遇到问题时，请在提交问题报告前收集日志：

```bash
# 导出最近 200 行日志
journalctl -u nonebot -n 200 > debug.log

# 提交 debug.log 并说明：
# 1. 命令是什么 (/search keyword 还是 /download keyword)
# 2. 什么时候执行的（时间）
# 3. 期望结果vs实际结果
```

## 禁用直连 IP 模式进行测试

如果怀疑直连 IP 有问题，可以临时禁用：

```env
EHENTAI_ENABLE_DIRECT_IP=false
```

重启服务后，日志应该直接显示 `[搜索] 使用标准 DNS 模式` 而不会出现 IP 相关的日志。
